# 大语言模型后训练：微调、对齐与评测综述

## 摘要

本报告对大语言模型（LLM）后训练技术进行了全面综述，涵盖三个关键领域：微调（Fine-tuning）、对齐（Alignment）和评测框架（Evaluation Harness）。我们考察了从全量微调到参数高效方法（如 LoRA 和 QLoRA）的演进，调研了包括 RLHF、DPO 和 GRPO 在内的对齐技术，并回顾了用于评估 LLM 能力的主要评测框架。本综述旨在提供对现代 LLM 后训练流程的实用理解。

**关键词**：大语言模型、微调、LoRA、QLoRA、RLHF、DPO、GRPO、评测框架

---

## 1. 引言

大语言模型（LLM）经历多阶段训练过程。初始的**预训练**阶段在大规模语料上训练模型以学习通用语言模式。然而，仅靠预训练模型不足以满足实际应用需求——它们可能生成无用、有害或不真实的输出。**后训练**是指将预训练模型转变为有用、安全且可靠的助手的后续阶段。

后训练流程通常包含三个阶段：

```
预训练模型 → 微调 → 对齐 → 评测
```

本报告详细调研了每个阶段，涵盖关键方法、权衡取舍以及用于实现和评估的工具。

---

## 2. 微调（Fine-tuning）

### 2.1 什么是微调？

微调是在预训练模型基础上，使用较小的、任务特定的数据集继续训练的过程。其目标是将模型的通用知识适配到特定领域或任务，而无需承担从头训练的高昂成本。

关键洞察在于：预训练 LLM 已经学习了丰富的语言表征——句法、语义、世界知识和推理模式。微调调整这些表征以在目标任务（如问答、代码生成或指令跟随）上表现更好。

### 2.2 预训练 vs 微调 vs 指令微调

这三个阶段在 LLM 训练流程中各有不同的目的：

| 方面 | 预训练 | 微调 | 指令微调 |
|------|--------|------|----------|
| **目标** | 下一个 token 预测 | 任务特定损失 | 响应生成 |
| **数据** | 大规模无标注文本 | 任务特定标注数据 | 指令-响应对 |
| **规模** | 数十亿 token | 数千到数百万 | 数千到数百万 |
| **算力** | 数千 GPU 小时 | 数小时到数天 | 数小时到数天 |
| **目的** | 学习语言 | 适配任务 | 跟随指令 |

**指令微调**是微调的一种特殊形式，模型在（指令、输入、输出）三元组上进行训练。这种方法由 Stanford Alpaca [6] 推广，教会模型跟随多样化的人类指令，而非执行单一任务。

### 2.3 全量微调 vs 参数高效微调

**全量微调**在训练过程中更新所有模型参数。对于使用 Adam 优化器的 7B 参数模型，这需要约 60-80 GB 的 GPU 显存（参数 + 梯度 + 优化器状态），对大多数研究者而言不太现实。

**参数高效微调（PEFT）**通过仅更新一小部分参数来解决这个问题。主要的 PEFT 方法包括：

- **LoRA**：权重矩阵的低秩适配 [1]
- **QLoRA**：量化基础模型上的 LoRA [2]
- **Prefix Tuning**：在注意力层前添加可训练向量 [4]
- **Adapter Tuning**：在层间插入可训练模块 [5]
- **Prompt Tuning**：学习软提示嵌入

### 2.4 LoRA：低秩适配

LoRA [1] 基于以下观察：微调过程中的权重更新往往具有低内在秩。LoRA 不更新完整的权重矩阵 W ∈ ℝ^{d×d}，而是将更新分解为两个低秩矩阵：

```
W' = W + ΔW = W + B × A
```

其中 A ∈ ℝ^{r×d}，B ∈ ℝ^{d×r}，秩 r << d。

**为什么有效**：对于维度 d=4096 的权重矩阵，全量微调需要更新 1670 万参数，而 r=16 的 LoRA 仅更新 13.1 万参数——减少 100 倍，性能损失极小。

**关键超参数**：
- **r（秩）**：控制表达能力，典型值：8-64
- **lora_alpha**：缩放因子，通常设为 2×r
- **target_modules**：适配哪些层（通常是注意力投影层）

**优势**：
- 可训练参数减少 10-100 倍
- 无额外推理延迟（权重可合并）
- 可为不同任务切换适配器

### 2.5 QLoRA：量化 LoRA

QLoRA [2] 通过将 4 位量化与 LoRA 结合，进一步提升显存效率：

1. **4 位 NormalFloat（NF4）**：使用信息论最优的量化方法将基础模型量化到 4 位，适用于正态分布的权重
2. **双重量化**：对量化常数本身进行量化，每个 65B 模型节省约 0.5 GB
3. **分页优化器**：使用 CPU 内存存放优化器状态以应对显存峰值

**7B 模型显存对比**：

| 方法 | GPU 显存 |
|------|----------|
| 全量微调（16 位） | ~60 GB |
| LoRA（16 位） | ~16 GB |
| QLoRA（4 位） | ~6 GB |

QLoRA 使得在单块 48GB GPU 上微调 65B 参数模型成为可能，让大模型微调更加普及。

### 2.6 Stanford Alpaca：案例研究

Stanford Alpaca [6] 证明了在相对较小但高质量的数据集上进行指令微调，可以显著提升 LLM 的指令跟随能力。

**方法**：
1. 基础模型：LLaMA-7B
2. 数据生成：使用 GPT-3.5-turbo 生成 5.2 万条指令跟随样本（约 $500）
3. 训练：标准监督微调（约 $100）
4. 结果：一个 7B 模型在指令跟随方面可媲美更大的模型

**Alpaca 数据格式**：
```json
{
  "instruction": "给出保持健康的三个建议。",
  "input": "",
  "output": "1. 均衡饮食。2. 规律运动。3. 保证充足睡眠。"
}
```

**影响**：Alpaca 证明了数据质量比数量更重要，启发了"自指令"范式，并催生了众多后续工作（Vicuna、WizardLM 等）。

### 2.7 对比：全量微调 vs LoRA vs QLoRA

| 方面 | 全量微调 | LoRA | QLoRA |
|------|----------|------|-------|
| 可训练参数 | 100% | ~0.1-1% | ~0.1-1% |
| GPU 显存（7B） | ~60 GB | ~16 GB | ~6 GB |
| 训练速度 | 基准 | 快 1.2-1.5 倍 | 0.8-1.0 倍 |
| 性能 | 最佳 | 接近全量微调 | 接近全量微调 |
| 推理延迟 | 基准 | 相同 | 相同 |
| 最低 GPU 要求 | A100 80GB | RTX 3090 24GB | T4 16GB |
| 适用场景 | 最大性能 | 资源受限 | 非常有限的 GPU |

**建议**：当 GPU 资源有限时，从 QLoRA 开始。它以极小的显存成本实现约 95% 的全量微调性能。

## 3. 对齐（Alignment）

### 3.1 什么是对齐？

对齐是指确保语言模型的行为与人类价值观和意图保持一致的过程。其目标是产生以下特性的模型：

- **有帮助（Helpful）**：提供有用、相关的响应
- **无害（Harmless）**：避免生成有毒、有偏见或危险的内容
- **诚实（Honest）**：提供真实信息，承认不确定性

预训练模型针对下一个 token 预测进行优化，而非成为有帮助的助手。即使经过指令微调，模型仍可能生成有害内容、产生幻觉或表现出不良行为。对齐训练正是为了解决这些问题。

### 3.2 指令微调 vs 对齐

指令微调和对齐服务于互补的目的：

| 方面 | 指令微调 | 对齐 |
|------|----------|------|
| 核心问题 | "模型能做 X 吗？" | "模型应该做 X 吗？" |
| 数据 | （指令、响应）对 | （优选、劣选）偏好对 |
| 方法 | 监督学习 | 强化学习或偏好优化 |
| 关注点 | 能力 | 价值观与安全 |

指令微调是对齐的前提——模型必须先具备跟随指令的能力，才能被对齐到人类偏好。

### 3.3 RLHF：基于人类反馈的强化学习

RLHF [10] 是主流的对齐范式，应用于 InstructGPT、ChatGPT 和 Claude。它包含三个阶段：

**阶段 1：监督微调（SFT）**
- 在高质量指令-响应对上微调基础模型
- 创建能够跟随指令的起点
- 通常使用数千到数万个样本

**阶段 2：奖励模型（RM）训练**
- 人类标注者对同一提示的多个模型输出进行排序
- 训练奖励模型以预测人类偏好
- 基于 Bradley-Terry 模型的损失函数：

```
L_RM = -E[log(σ(r(x, y_w) - r(x, y_l)))]
```

其中 y_w 是优选响应，y_l 是劣选响应。

**阶段 3：强化学习优化（PPO）**
- 使用奖励模型作为奖励信号
- 优化策略（LLM）以最大化期望奖励
- 包含 KL 散度惩罚以防止偏离 SFT 模型：

```
L_PPO = E[r(x, y) - β × KL(π_θ || π_ref)]
```

### 3.4 PPO 在 RLHF 中的应用

PPO（近端策略优化）[15] 用于 RLHF 的阶段 3，因为标准策略梯度方法不稳定。PPO 通过裁剪目标函数来防止过大的更新：

```
L_PPO = E[min(r_t(θ) × A_t, clip(r_t(θ), 1-ε, 1+ε) × A_t)]
```

其中 r_t(θ) 是概率比率，A_t 是优势估计，ε 是裁剪参数。

**PPO-based RLHF 的挑战**：
- **奖励黑客（Reward Hacking）**：模型找到绕过真正有帮助来最大化奖励的方式
- **训练不稳定性**：需要仔细调整超参数
- **计算成本**：同时维护 4 个模型（策略、参考、奖励、价值）
- **KL 约束调优**：β 需要仔细平衡

### 3.5 DPO：直接偏好优化

DPO [12] 通过直接在偏好数据上优化策略，消除了对独立奖励模型的需求。

**核心洞察**：RLHF 下的最优策略具有闭式解，可以用策略本身来表达奖励：

```
r(x,y) = β × log(π*(y|x) / π_ref(y|x)) + β × log(Z(x))
```

由此得到 DPO 损失：

```
L_DPO = -E[log(σ(β × (log(π_θ(y_w|x)/π_ref(y_w|x)) - log(π_θ(y_l|x)/π_ref(y_l|x))))]
```

**相比 RLHF 的优势**：
- 无需奖励模型
- 无需强化学习训练循环
- 更简单、更稳定的训练
- 更低的计算成本

**局限性**：
- 需要离线偏好数据
- 无法进行在线探索
- 在某些场景下可能不如 RLHF

### 3.6 RLAIF：基于 AI 反馈的强化学习

RLAIF [13] 用 AI 模型替代人类标注者来生成偏好数据：

1. 为同一提示生成多个响应
2. 使用强大的 AI 模型（如 GPT-4）对响应进行排序/评估
3. 在 AI 生成的偏好数据上训练

**优势**：可扩展、一致、成本低。**局限性**：继承 AI 评判者的偏见，可能无法捕捉细微的人类价值观。Anthropic 的 Constitutional AI [13] 是一种相关方法，使用带有明确原则的 AI 反馈。

### 3.7 GRPO：群组相对策略优化

GRPO [14] 是 DeepSeek-R1 中使用的较新对齐方法，通过移除评论家/价值模型来简化 PPO：

1. 对每个提示，从当前策略中采样 G 个响应
2. 用奖励函数对每个响应评分
3. 通过组内归一化计算优势：

```
A_i = (r_i - mean(r_1...r_G)) / std(r_1...r_G)
```

4. 使用裁剪目标更新策略

**优势**：无需价值模型（节省约 50% 显存），实现更简单，对具有可验证奖励的推理任务有效。

### 3.8 对齐与推理

对齐可以通过以下方式提升推理能力：
- **思维链引导**：教会模型"逐步思考"
- **过程奖励模型（PRM）**：奖励中间推理步骤
- **结果奖励模型（ORM）**：奖励最终答案的正确性
- **可验证奖励**：使用数学/代码正确性作为奖励信号

GRPO 结合可验证奖励在推理任务上表现出特别的有效性，DeepSeek-R1 在数学和编码基准上的强劲表现证明了这一点。

### 3.9 对齐方法总结

| 方法 | 奖励模型 | RL 训练 | 复杂度 | 最佳适用场景 |
|------|----------|---------|--------|-------------|
| RLHF/PPO | 是 | 是 | 高 | 通用对齐 |
| DPO | 否 | 否 | 低 | 离线偏好 |
| RLAIF | AI 评判 | 否 | 低 | 可扩展对齐 |
| GRPO | 否 | 简化 | 中 | 推理任务 |

## 4. 评测框架（Evaluation Harness）

### 4.1 什么是评测框架？

评测框架是一个标准化的系统，用于在多个基准测试、任务和指标上系统地评估语言模型。它为 LLM 评测提供了一致性、可复现性、可比性和自动化能力。

没有标准化评测，不同论文使用不同的设置，无法公平比较模型。评测框架通过提供固定协议来解决这个问题。

### 4.2 核心概念

**基准测试（Benchmark）**是测试特定能力的任务集合：
- **MMLU** [18]：多任务语言理解（57 个学科）
- **HellaSwag** [19]：常识推理
- **ARC** [20]：科学问答
- **GSM8K** [21]：小学数学
- **HumanEval** [22]：代码生成
- **TruthfulQA**：真实性
- **C-Eval/CMMLU** [23,24]：中文基准测试

**评估指标**因任务而异：
- **准确率（Accuracy）**：精确匹配（分类任务）
- **困惑度（Perplexity）**：语言建模质量
- **Pass@k**：代码生成正确率
- **BLEU/ROUGE**：文本生成质量

**少样本设置**：
- **零样本（Zero-shot）**：提示中无示例
- **少样本（Few-shot/k-shot）**：提供 k 个示例（通常 1-5 个）

### 4.3 EleutherAI lm-evaluation-harness

lm-evaluation-harness [26] 是最广泛使用的开源评测框架，由 EleutherAI 开发。

**特点**：200+ 任务、多种模型后端（HuggingFace、vLLM、OpenAI API）、可配置少样本设置、标准化指标、多 GPU 支持。

**使用方法**：
```bash
# 基础评测
lm_eval --model hf \
    --model_args pretrained=Qwen/Qwen2.5-0.5B \
    --tasks hellaswag,mmlu \
    --device cuda:0 \
    --batch_size 8

# 少样本评测
lm_eval --model hf \
    --model_args pretrained=MODEL_NAME \
    --tasks mmlu \
    --num_fewshot 5 \
    --device cuda:0

# 完整排行榜评测
lm_eval --model hf \
    --model_args pretrained=MODEL_NAME \
    --tasks mmlu,hellaswag,arc_challenge,winogrande,truthfulqa,gsm8k \
    --output_path results/
```

### 4.4 Hugging Face 评测工具

Hugging Face 提供了互补的评测工具：

**evaluate 库**：计算指标的简单 API：
```python
from evaluate import load
accuracy = load("accuracy")
results = accuracy.compute(references=[0, 1, 2], predictions=[0, 1, 1])
```

**Open LLM Leaderboard**：使用标准化基准测试对开源 LLM 进行公开排名，由 lm-evaluation-harness 驱动。

### 4.5 OpenCompass

OpenCompass [25] 是由上海人工智能实验室开发的综合评测平台：
- 100+ 数据集覆盖各种能力
- 模块化评测架构
- 强大的中文基准测试覆盖（C-Eval、CMMLU、GAOKAO）
- 分布式评测支持

```bash
python run.py --models hf_llama2_7b --datasets mmlu_ppl
```

### 4.6 HELM

HELM（语言模型整体评估）[17] 由 Stanford CRFM 开发，提供多维度评估：
- 在 7 个维度上评估：准确率、校准、鲁棒性、公平性、偏见、毒性、效率
- 42 个场景覆盖多样化能力
- 标准化的提示、指标和评估协议

```bash
helm-run --run-entries mmlu:model=openai/gpt-3.5-turbo --suite my-suite
helm-summarize --suite my-suite
```

### 4.7 评测最佳实践

1. **选择合适的基准测试**以匹配你的使用场景
2. **使用一致的设置**：相同的少样本数量、提示模板、批大小
3. **报告置信区间**：始终包含标准误差
4. **注意数据污染**：训练数据可能与基准测试重叠
5. **公平比较**：所有模型使用相同的评测设置

## 5. 后训练流程

完整的 LLM 后训练流程如下：

```
预训练模型（如 LLaMA、Qwen）
    │
    ▼
监督微调（SFT）
    │  - 指令跟随数据
    │  - 使用 LoRA / QLoRA 提高效率
    │
    ▼
偏好数据收集
    │  - 人类标注或 AI 反馈
    │
    ▼
对齐训练
    │  - RLHF/PPO、DPO 或 GRPO
    │
    ▼
对齐后的模型
    │
    ▼
评测
    │  - lm-evaluation-harness、OpenCompass、HELM
    │
    ▼
基准测试结果与错误分析
```

每个阶段建立在前一阶段的基础上，逐步将原始语言模型转变为有能力、安全且可靠的助手。

---

## 6. 结论

本综述涵盖了 LLM 后训练的三大支柱：

1. **微调**将预训练模型适配到特定任务，LoRA 和 QLoRA 以最小成本实现了高效适配
2. **对齐**确保模型按照人类价值观行事，DPO 为传统 RLHF 提供了更简单的替代方案
3. **评测框架**为衡量模型能力提供了标准化系统

该领域正在快速发展，GRPO 等新方法正在推动对齐技术的边界，特别是在推理任务方面。随着 LLM 能力的持续增长，稳健的后训练和评测对于确保其安全部署将变得越来越重要。
