# 方法对比表

## 1. 微调方法对比

### 全量微调 vs LoRA vs QLoRA

| 方面 | 全量微调 | LoRA | QLoRA |
|------|----------|------|-------|
| **可训练参数** | 100%（全部参数） | ~0.1-1% | ~0.1-1% |
| **GPU 显存（7B）** | ~60 GB | ~16 GB | ~6 GB |
| **GPU 显存（13B）** | ~120 GB | ~30 GB | ~12 GB |
| **GPU 显存（65B）** | ~500 GB | ~120 GB | ~48 GB |
| **训练速度** | 基准 | 快 1.2-1.5 倍 | 0.8-1.0 倍（反量化开销） |
| **性能** | 最佳 | 全量微调的 ~98% | 全量微调的 ~95% |
| **推理延迟** | 基准 | 相同（合并权重） | 相同（合并权重） |
| **多任务支持** | 独立模型 | 切换适配器 | 切换适配器 |
| **最低 GPU 要求** | A100 80GB | RTX 3090 24GB | T4 16GB |
| **实现方式** | 标准训练 | PEFT 库 | PEFT + bitsandbytes |
| **最佳适用场景** | 最大性能 | 资源受限 | 非常有限的 GPU |

### 关键要点
- **全量微调**：性能最好，但大模型的显存成本过高
- **LoRA**：接近全量微调性能，显存减少 4 倍，无推理开销
- **QLoRA**：使得在消费级 GPU 上微调 65B 模型成为可能，约 95% 的全量微调性能

## 2. 对齐方法对比

### RLHF/PPO vs DPO vs RLAIF vs GRPO

| 方面 | RLHF/PPO | DPO | RLAIF | GRPO |
|------|----------|-----|-------|------|
| **奖励模型** | 是（训练的） | 否 | AI 评判 | 否（基于群组） |
| **RL 训练** | 是（PPO） | 否 | 否 | 是（简化） |
| **训练复杂度** | 高 | 低 | 低 | 中 |
| **需要的模型数** | 4 个（策略、参考、RM、价值） | 2 个（策略、参考） | 2 个（策略、参考） | 2 个（策略、参考） |
| **GPU 显存** | 非常高 | 中等 | 中等 | 高 |
| **数据类型** | 偏好对 | 偏好对 | AI 排序对 | 可验证奖励 |
| **在线探索** | 是 | 否 | 否 | 是 |
| **稳定性** | 中 | 高 | 高 | 中 |
| **性能** | 强 | 良好 | 良好 | 强（推理） |
| **最佳适用** | 通用对齐 | 离线偏好 | 可扩展对齐 | 推理任务 |

### RLHF 流程
```
SFT → 奖励建模 → PPO → 对齐后的模型
```

### DPO 流程
```
SFT → 偏好数据 → DPO → 对齐后的模型
```

### GRPO 流程
```
SFT → 面向推理的 RL → GRPO → 对齐后的模型
```

## 3. 评测框架对比

### lm-evaluation-harness vs OpenCompass vs HELM

| 方面 | lm-eval-harness | OpenCompass | HELM |
|------|----------------|-------------|------|
| **开发者** | EleutherAI | 上海人工智能实验室 | Stanford CRFM |
| **任务/数据集** | 200+ | 100+ | 42 个场景 |
| **中文基准测试** | 有限 | 强（C-Eval、CMMLU） | 有限 |
| **模型后端** | HF、vLLM、OpenAI 等 | HF、API 模型 | HF、API 模型 |
| **易用性** | 高 | 中 | 中 |
| **文档** | 优秀 | 良好 | 良好 |
| **社区** | 非常活跃 | 活跃 | 活跃 |
| **主要用途** | 标准评估 | 中文 + 综合 | 整体评估 |
| **输出格式** | JSON | JSON、表格 | HTML 报告 |
| **多 GPU** | 是 | 是 | 是 |

### 何时使用哪个框架
- **lm-evaluation-harness**：大多数评估的标准选择，特别是英文
- **OpenCompass**：最适合中文基准测试，综合多维度评估
- **HELM**：最适合跨多维度的整体评估（公平性、偏见、毒性）

## 5. LoRA 超参数指南

| 参数 | 典型范围 | 作用 | 建议 |
|------|----------|------|------|
| **r（秩）** | 4-64 | 越大 = 表达能力越强 | 从 16 开始 |
| **lora_alpha** | 16-64 | 缩放因子 | 设为 2×r |
| **lora_dropout** | 0.0-0.1 | 正则化 | 0.05 |
| **target_modules** | 注意力层 | 适配哪些层 | q_proj, v_proj, k_proj, o_proj |
| **learning_rate** | 1e-4 到 3e-4 | 收敛速度 | 2e-4 |
| **epochs** | 1-3 | 训练时长 | 2-3 |
