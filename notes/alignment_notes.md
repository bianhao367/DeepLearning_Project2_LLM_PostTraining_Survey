# 对齐笔记

## 1. 什么是对齐？

对齐是指确保语言模型的行为与人类价值观和意图保持一致的过程。其目标是产生具有以下特性的模型：

- **有帮助（Helpful）**：提供有用、相关的响应
- **无害（Harmless）**：避免生成有毒、有偏见或危险的内容
- **诚实（Honest）**：提供真实信息，承认不确定性

### 为什么对齐很重要
- 预训练模型针对下一个 token 预测优化，而非成为有帮助的助手
- 它们可能生成有害、有偏见或事实不正确的内容
- 仅靠指令微调是不够的——模型需要显式的对齐训练

## 2. 指令微调 vs 对齐

| 方面 | 指令微调 | 对齐 |
|------|----------|------|
| **目标** | 跟随指令 | 跟随人类价值观 |
| **数据** | （指令、响应）对 | （优选、劣选）偏好对 |
| **方法** | 监督学习 | 强化学习或偏好优化 |
| **解决的问题** | "模型能做 X 吗？" | "模型应该做 X 吗？" |

指令微调是对齐的**前提**——模型必须先具备跟随指令的能力，才能被对齐到人类偏好。

## 3. RLHF（基于人类反馈的强化学习）

### 概述
RLHF 是主流的对齐范式，应用于 InstructGPT、ChatGPT 和 Claude。它包含三个阶段：

```
阶段 1：监督微调（SFT）
    ↓
阶段 2：奖励模型训练
    ↓
阶段 3：强化学习优化（PPO）
```

### 阶段 1：SFT
- 在高质量指令-响应对上微调基础模型
- 创建能够跟随指令的"起点"
- 通常使用数千到数万个样本

### 阶段 2：奖励模型（RM）
- 人类标注者对同一提示的多个模型输出进行排序
- 训练奖励模型以预测人类偏好
- 架构：与策略模型相同，但输出标量奖励
- 损失：Bradley-Terry 模型——最大化优选响应相对于劣选响应的对数概率

```
L_RM = -E[log(σ(r(x, y_w) - r(x, y_l)))]
```

其中 y_w 是优选响应，y_l 是劣选响应。

### 阶段 3：PPO（近端策略优化）
- 使用奖励模型作为奖励信号
- 优化策略（LLM）以最大化期望奖励
- 包含 KL 散度惩罚以防止偏离 SFT 模型

```
L_PPO = E[r(x, y) - β × KL(π_θ || π_ref)]
```

其中 β 控制 KL 惩罚的强度。

## 4. PPO 详解

### 为什么用 PPO？
- 标准策略梯度方法不稳定
- PPO 通过裁剪目标函数来防止过大的更新
- 稳定且在实践中广泛使用

### PPO 目标函数
```
L_PPO = E[min(r_t(θ) × A_t, clip(r_t(θ), 1-ε, 1+ε) × A_t)]
```

其中：
- r_t(θ) = 新策略概率与旧策略概率的比率
- A_t = 优势估计
- ε = 裁剪参数（通常为 0.1-0.2）

### PPO 的挑战
- **奖励黑客（Reward Hacking）**：模型找到绕过真正有帮助来最大化奖励的方式
- **训练不稳定性**：需要仔细调整超参数
- **计算成本**：需要同时维护 4 个模型（策略、参考、奖励、价值）
- **KL 约束调优**：β 需要仔细平衡

## 5. DPO（直接偏好优化）

### 核心思想
DPO 通过直接在偏好数据上优化策略，消除了对独立奖励模型的需求。

### 数学基础
关键洞察是：RLHF 下的最优策略具有闭式解：

```
π*(y|x) = π_ref(y|x) × exp(r(x,y)/β) / Z(x)
```

重排以表达奖励：

```
r(x,y) = β × log(π*(y|x) / π_ref(y|x)) + β × log(Z(x))
```

这意味着我们可以用策略本身来表达奖励，从而消除对独立奖励模型的需求。

### DPO 损失
```
L_DPO = -E[log(σ(β × (log(π_θ(y_w|x)/π_ref(y_w|x)) - log(π_θ(y_l|x)/π_ref(y_l|x))))]
```

### 相比 RLHF 的优势
- 无需奖励模型
- 无需强化学习训练循环
- 更简单、更稳定的训练
- 单阶段优化
- 更低的计算成本

### 局限性
- 需要离线偏好数据
- 无法进行在线探索
- 在某些场景下可能不如 RLHF

## 6. RLAIF（基于 AI 反馈的强化学习）

### 核心思想
用 AI 模型替代人类标注者来生成偏好数据。

### 流程
1. 为同一提示生成多个响应
2. 使用强大的 AI 模型（如 GPT-4）对响应进行排序/评估
3. 在 AI 生成的偏好数据上训练

### 优势
- 可扩展——无人类标注瓶颈
- 一致性——AI 偏好比人类更一致
- 成本效益——比人类标注便宜得多

### 局限性
- 继承 AI 评判者的偏见
- 可能无法捕捉细微的人类价值观
- "Constitutional AI"是相关方法（Anthropic）

## 7. GRPO（群组相对策略优化）

### 概述
GRPO 是 DeepSeek-R1 中使用的较新对齐方法。它通过以下方式简化 PPO：
1. 移除评论家/价值模型
2. 使用基于群组的优势估计
3. 相对于一组采样响应计算优势

### 工作原理
1. 对每个提示，从当前策略中采样 G 个响应
2. 用奖励函数对每个响应评分
3. 通过组内归一化计算优势：

```
A_i = (r_i - mean(r_1...r_G)) / std(r_1...r_G)
```

4. 使用裁剪目标更新策略（类似 PPO）

### 优势
- 无需价值/评论家模型（节省约 50% 显存）
- 实现更简单
- 对推理任务（数学、代码）有效
- 更适合具有可验证奖励的任务

### GRPO 在 DeepSeek-R1 中的应用
- 用于训练推理能力
- 奖励函数基于正确性（如数学答案验证）
- 在数学和编码基准上显示出显著改进

## 8. 对齐与推理

### 对齐如何提升推理能力
- **思维链引导**：对齐可以教会模型"逐步思考"
- **过程奖励模型（PRM）**：奖励中间推理步骤，而非仅奖励最终答案
- **自我纠正**：对齐后的模型可以学会验证和纠正其推理

### 面向推理的对齐
- GRPO 结合可验证奖励（数学/代码正确性）
- 过程奖励模型（PRM）用于逐步验证
- 结果奖励模型（ORM）用于最终答案验证

## 9. 总结：对齐方法对比

| 方法 | 奖励模型 | RL 训练 | 复杂度 | 性能 |
|------|----------|---------|--------|------|
| RLHF/PPO | 是 | 是 | 高 | 强 |
| DPO | 否 | 否 | 低 | 良好 |
| RLAIF | AI 评判 | 否 | 低 | 良好 |
| GRPO | 否（基于群组） | 简化 | 中 | 强（推理） |
