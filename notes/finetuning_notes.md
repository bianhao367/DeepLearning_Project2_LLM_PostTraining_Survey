# 微调笔记

## 1. 什么是微调？

微调是在预训练模型基础上，使用较小的、任务特定的数据集继续训练的过程。其目标是将模型的通用知识适配到特定领域或任务，而无需从头训练。

**核心思想**：预训练 LLM 已经学习了语言模式、句法和世界知识。微调调整其参数以在目标任务（如问答、代码生成、指令跟随）上表现更好。

## 2. 预训练 vs 微调 vs 指令微调

| 方面 | 预训练 | 微调 | 指令微调 |
|------|--------|------|----------|
| **目标** | 学习通用语言 | 适配特定任务 | 跟随用户指令 |
| **数据** | 大规模无标注文本 | 任务特定标注数据 | 指令-响应对 |
| **目标函数** | 下一个 token 预测（CLM） | 任务特定损失（如交叉熵） | 响应生成损失 |
| **算力** | 非常高（数千 GPU 小时） | 中等 | 中等 |
| **示例** | GPT-4 在互联网文本上预训练 | 医疗问答微调 | Alpaca 风格指令跟随 |

**指令微调**是微调的一种特殊形式，模型在（指令、输入、输出）三元组上训练，以提升在多样化任务中跟随人类指令的能力。

## 3. 全量微调 vs 参数高效微调（PEFT）

### 全量微调
- 更新**所有**模型参数
- 需要大量 GPU 显存（模型参数 + 梯度 + 优化器状态）
- 对于 7B 模型：约 60-80 GB GPU 显存（使用 Adam 优化器）
- 性能最好但成本最高

### 参数高效微调（PEFT）
- 仅更新参数的**小子集**
- 显存占用大幅降低
- 主要方法包括：
  - **LoRA**：权重矩阵的低秩适配
  - **QLoRA**：量化基础模型上的 LoRA
  - **Prefix Tuning**：在注意力层前添加可训练向量
  - **Adapter Tuning**：在层间插入小型可训练模块
  - **Prompt Tuning**：学习软提示嵌入

## 4. LoRA（低秩适配）

### 核心思想
不更新完整的权重矩阵 W（d×d），而是将更新分解为两个低秩矩阵：

```
W' = W + ΔW = W + B × A
```

其中：
- A ∈ R^{r×d}（下投影）
- B ∈ R^{d×r}（上投影）
- r << d（秩，通常为 4-64）

### 为什么有效
- 微调过程中的权重更新往往具有低内在秩
- 只需训练 r×d + d×r 个参数，而非 d×d 个
- 对于 d=4096, r=16：4096×16×2 = 131K vs 4096×4096 = 1670 万参数

### 关键超参数
- **r（秩）**：越大 = 表达能力越强，参数越多。典型值：8-64
- **lora_alpha**：缩放因子，通常设为 2×r
- **lora_dropout**：LoRA 层的 Dropout 率
- **target_modules**：应用 LoRA 的层（如 q_proj, v_proj）

### 优势
- 可训练参数减少 10-100 倍
- 无额外推理延迟（可合并权重）
- 可为不同任务切换适配器

## 5. QLoRA（量化 LoRA）

### 核心思想
将基础模型的 4 位量化与 LoRA 微调相结合：

1. **4 位 NormalFloat（NF4）**：使用信息论最优的量化方法将基础模型量化到 4 位
2. **双重量化**：对量化常数本身进行量化（每个 65B 模型节省约 0.5 GB）
3. **分页优化器**：使用 CPU 内存存放优化器状态以应对显存峰值

### 显存节省
| 模型 | 全量微调（16 位） | LoRA（16 位） | QLoRA（4 位） |
|------|-------------------|---------------|---------------|
| 7B   | ~60 GB            | ~16 GB        | ~6 GB         |
| 13B  | ~120 GB           | ~30 GB        | ~12 GB        |
| 65B  | ~500 GB           | ~120 GB       | ~48 GB        |

### 关键创新
- NF4 量化对正态分布权重是信息论最优的
- 通过"分块量化"实现量化权重的反向传播
- 使得在单块 48GB GPU 上微调 65B 模型成为可能

## 6. Stanford Alpaca 案例研究

### 概述
Stanford Alpaca 证明了在相对较小的数据集上进行指令微调，可以显著提升 LLM 的指令跟随能力。

### 方法
1. **基础模型**：LLaMA-7B
2. **数据生成**：使用 GPT-3.5-turbo 生成 5.2 万条指令跟随样本
3. **训练**：使用标准监督学习微调
4. **成本**：数据生成约 $500，训练约 $100

### Alpaca 数据格式
```json
{
  "instruction": "给出保持健康的三个建议。",
  "input": "",
  "output": "1. 均衡饮食。2. 规律运动。3. 保证充足睡眠。"
}
```

### 影响
- 证明了数据质量比数量更重要
- 启发了许多后续工作（Vicuna、WizardLM 等）
- 展示了"自指令"范式：使用 LLM 生成训练数据

## 7. 对比：全量微调 vs LoRA vs QLoRA

| 方面 | 全量微调 | LoRA | QLoRA |
|------|----------|------|-------|
| **可训练参数** | 100% | ~0.1-1% | ~0.1-1% |
| **GPU 显存（7B）** | ~60 GB | ~16 GB | ~6 GB |
| **训练速度** | 基准 | 快 1.2-1.5 倍 | 0.8-1.0 倍（反量化开销） |
| **性能** | 最佳 | 接近全量微调 | 接近全量微调 |
| **推理延迟** | 基准 | 相同（合并权重） | 相同（合并权重） |
| **多任务** | 独立模型 | 切换适配器 | 切换适配器 |
| **最低 GPU** | A100 80GB | RTX 3090 24GB | RTX 3090 24GB / T4 16GB |
| **适用场景** | 最大性能，充足资源 | 资源受限，多任务 | 非常有限的 GPU 资源 |

## 8. 实用建议

1. **从 QLoRA 开始**：如果 GPU 显存有限——它能达到约 95% 的全量微调性能
2. **学习率**：LoRA 通常为 1e-4 到 3e-4（高于全量微调）
3. **训练轮次**：指令微调通常 1-3 轮即可
4. **数据质量 > 数量**：1000 条高质量样本通常优于 10000 条噪声数据
5. **监控损失**：注意过拟合——指令微调数据量较小
